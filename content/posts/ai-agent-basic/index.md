## 什么是 AI Agent
AI Agent 是一种能够 **感知环境、进行思考与制定计划、并采取行动来实现特定目标** 的计算机程序或系统。
它通过摄像头、麦克风或软件输入等方式来获取外部信息，对这些信息进行分析与判断，并基于掌握的信息与既定目标选择最合适的行动方案。
在行动执行完毕后，Agent 会从结果中总结经验，用于优化后续的决策过程。

常见的 AI Agent 包括：聊天机器人、自动驾驶系统、游戏中的 NPC 等。

## 理解 AI 系统的不同层级
### Level 1:大语言模型(LLM)
LLM 是许多 AI 系统的基础层，其核心能力是理解用户输入，并结合训练过程中学到的知识生成自然语言文本。

尽管 LLM 本身能力强大，但仍然存在明显局限：
1. **数据访问受限**：在未与外部系统集成的情况下，LLM 无法访问实时数据或私有数据，难以提供最新或高度专业化的信息
2. **被动相应**：LLM 通常只会根据用户输入执行一次性回应，在没有下一步的明确指令前，它不会主动持续推进任务

总体来看，LLM 更像是一个强大的文本生成引擎，缺乏跨系统的信息整合能力与自主行为能力。

### Level 2:AI 工作流
AI 工作流是在 LLM 之上，通过预定义好的步骤，将多个操作串联成一条自动化流程，主要用于简化重复性、结构清晰且可预测的任务。

然而，AI 工作流同样存在限制：
1. **逻辑依赖人工预设、灵活性有限**：工作流的执行路径由人为规则决定，当出现预期之外的情况时，通常需要人工介入进行调整。

### Level 3:AI Agent
AI Agent 代表了当前 AI 系统中最高级的形态，它能够自主进行推理、执行行动，并基于结果不断迭代优化自身行为，整个过程无需持续的人为指导。

AI Agent 的核心特征包括：
1. **推理（Reasoning）**：分析当前状态和上下文，判断实现目标的最佳策略
2. **行动（Acting）**：调用工具、API 或其他资源，将决策转换为实际行动
3. **迭代（Iterating）**：根据行动结果进行反思与调整，持续优化后续行为

正式这种自适应与持续学习的能力，使 AI Agent 特别适合应对动态、复杂且不确定的场景。

## 支撑 AI Agent 的关键框架
AI Agent 的高级能力离不开底层框架的支持，其中最重要的两个是：
1. **检索增强生成（Retrieval-Augmented Generation，RAG）**：通过引入外部数据源（如企业内部数据库、实时信息系统等），使 Agent 能够获取私有或最新信息，从而提升行动的准确性与时效性
2. **ReAct 框架**：将“推理（Reasoning）”与“行动（Acting）”紧密结合，使 Agent 能够在思考的同时规划并执行任务

## 什么是工具(Tools)
工具是 AI Agent 在执行任务时可以调用的外部能力或资源，例如：网页搜索 API、计算器、数据库、语言翻译引擎等。
Agent 会向工具发送请求，获取结果，并将这些结果作为新的输入继续执行任务。

工具的引入，使体量较小或能力受限的 Agent 也可以完成原本难以胜任的复杂工作。
同时，工具还能显著提升 Agent 在时效性与准确性方面的表现。

如何选择合适的工具，以及在何时调用它们，是构建高质量 AI Agent 的关键因素之一。

## Agent 循环(Agent Loop)
Agent 循环是 AI Agent 持续向目标推进的核心机制。

在每一轮循环中，Agent 会先从记忆或工具中获取最新信息，然后更新自身状态，通过推理决定下一步行动，并执行相应操作。
行动完成后，Agent 会检查执行结果，将新信息存入记忆，随后进入下一轮循环。

正是这一机制，使 Agent 能够根据环境变化不断调整策略，并随着时间推移持续优化表现。

### 感知/用户输入
**感知（Perception）**，也称为 **用户输入（User Input）**，是 Agent 循环中的第一阶段。

Agent 会从外部世界收集信息，包括用户输入的文本、语音、图像或各类传感器数据。
随后，Agent 可能会对这些原始数据进行预处理，例如文本清洗、语音转文本、图像压缩或噪声过滤，以将其转换为可理解、可利用的形式。

良好的感知能力，意味着 Agent 可以在后续决策中基于真实、可靠的信息；反之，若输入本身存在偏差，后续推理与行动也可能随之失败。

### 推理与规划
**推理与规划（Reason and Plan）** 是 Agent 在采取行动前的关键阶段。

在这一阶段，Agent 会结合当前状态与目标，分析已有信息，并思考“为了实现目标，下一步应该做什么”。
通常，Agent 会将整体目标拆解为多个可执行的小步骤，对这些步骤进行合理性检查，并按顺序组织成清晰的行动计划。
同时，Agent 还可能预判潜在风险，并准备相应的备选方案。只有当整体计划足够可行时，Agent 才会进入执行阶段。

### 行动/工具调用
**行动（Acting）**，也称为 **工具调用（Tool Invocation）**，是 Agent 将计划转化为现实影响的阶段。

Agent 会根据既定目标与当前计划，选择合适的工具，并填入正确的参数发起调用。
具体的执行工作由外部系统完成，Agent 只需接收返回的结果即可。

### 观察与反思
**观察与反思（Observation & Reflection）** 是 Agent 在每一轮循环中的“暂停思考”阶段。

Agent 会先观察环境变化，收集最新信息，然后思考“这些变化对当前目标意味着什么”。
在此过程中，Agent 会更新自身记忆、识别潜在错误，并判断哪些信息最为关键。
这些反思结果将直接影响下一轮的推理规划与行动。若缺乏这一环节，Agent 容易依赖过时或错误的信息，最终逐渐偏离原本的目标。

## 总结
AI Agent 并非单一模型或技术，而是一种由多层能力共同构成的智能系统。
从只能被动生成文本的大语言模型，到依赖预设逻辑的 AI 工作流，再到具备自主推理、行动与迭代能力的 AI Agent，
AI 系统正逐步从“工具”演进为“能够自主完成任务的执行者”。

AI Agent 的核心在于闭环能力：通过感知获取信息，通过推理与规划制定行动方案，借助工具影响外部世界，并在观察与反思中不断修正自身行为。
这一循环机制使 Agent 能够在动态环境中持续逼近目标，而不需要依赖持续的人工干预。

总体而言，理解 AI Agent 不仅有助于正确使用现有的 AI 产品，也为设计更复杂、更可靠的智能系统提供了清晰的思路。
